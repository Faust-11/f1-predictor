# UI Guide

> Этот файл — единственный источник правды по визуальному стилю. Cursor не "понимает" слова вроде "в стиле Formula 1" — он работает только с конкретными значениями ниже. При любой стилизации компонента — сначала проверить этот файл, не придумывать цвета/радиусы на месте.

---

## 1. Цвета (CSS-переменные для shadcn/ui)

Вставить в `app/globals.css`. Значения в HSL (формат, который ожидает shadcn).

```css
:root {
  --background: 0 0% 100%;
  --foreground: 0 0% 9%;

  --primary: 358 100% 44%;        /* акцентный красный, в духе F1-брендинга */
  --primary-foreground: 0 0% 100%;

  --secondary: 0 0% 96%;
  --secondary-foreground: 0 0% 9%;

  --muted: 0 0% 92%;
  --muted-foreground: 0 0% 45%;

  --destructive: 0 84% 60%;
  --destructive-foreground: 0 0% 100%;

  --border: 0 0% 88%;
  --input: 0 0% 88%;
  --ring: 358 100% 44%;

  --radius: 0.75rem;
}

.dark {
  --background: 0 0% 7%;
  --foreground: 0 0% 98%;

  --primary: 358 100% 50%;
  --primary-foreground: 0 0% 100%;

  --secondary: 0 0% 14%;
  --secondary-foreground: 0 0% 98%;

  --muted: 0 0% 18%;
  --muted-foreground: 0 0% 65%;

  --destructive: 0 70% 50%;
  --destructive-foreground: 0 0% 100%;

  --border: 0 0% 20%;
  --input: 0 0% 20%;
  --ring: 358 100% 50%;
}
```

Подкорректируйте `--primary` под свой вкус — `358 100% 44%` это насыщенный красный, близкий по ощущению к брендингу F1, но не претендующий на точное совпадение с официальным.

## 2. Типографика

- **Заголовки:** "Titillium Web" (Google Fonts) — геометричный, технологичный шрифт, похожий на графику телетрансляций гонок.
- **Текст:** "Inter" — для читаемости длинных списков (рейтинг, история).

```ts
// app/layout.tsx
import { Titillium_Web, Inter } from "next/font/google";

const heading = Titillium_Web({ subsets: ["latin", "cyrillic"], weight: ["600", "700"], variable: "--font-heading" });
const body = Inter({ subsets: ["latin", "cyrillic"], variable: "--font-body" });
```

> Важно: подключить `cyrillic`/кириллический subset, интерфейс на украинском.

## 3. Радиусы и тени

| Токен | Значение | Где использовать |
|---|---|---|
| `rounded-sm` | 0.375rem | бейджи, чипы статуса |
| `rounded-md` (= `--radius`) | 0.75rem | карточки пилотов/команд, кнопки |
| `rounded-lg` | 1rem | баннер на главной, модальные окна |
| `rounded-full` | — | аватары пилотов, круглые иконки |

Тени — использовать стандартные Tailwind `shadow-sm` / `shadow-md`, не придумывать кастомные. На hover карточек — переход на `shadow-md` с `transition-shadow duration-200`.

---

## 4. Отступы

Использовать стандартную шкалу Tailwind (база 4px) без кастомизации — этого достаточно для всех макетов в проекте, дополнительная шкала не нужна.

---

## 5. Breakpoints

Стандартные Tailwind: `sm` 640px, `md` 768px, `lg` 1024px, `xl` 1280px. Дополнительно проверять вручную мобильную раскладку на 320px (минимальная ширина из ТЗ) — это уже ниже `sm`, верстать mobile-first без брейкпоинта для этой ширины.

---

## 6. Иконки

`lucide-react` — уже используется в shadcn/ui, не подключать вторую иконочную библиотеку.

---

## 7. Анимации (Framer Motion)

| Действие | Длительность | Easing |
|---|---|---|
| Hover/tap на карточке | 150–200ms | `easeOut` |
| Пилот "перелетает" в слот прогноза | 300–400ms | `easeOut`, с небольшим scale (0.95 → 1) |
| Появление секции при скролле | 300ms | `easeOut`, fade + translateY(8px) |
| Открытие модального окна | 200ms | fade + scale (0.96 → 1) |

Не использовать анимации дольше 500ms — на мобильных это ощущается как лаг, а не как "плавность".

---

## 8. Состояния компонентов (визуально)

- **Loading:** скелетон (серые прямоугольники с пульсацией, `animate-pulse`), не спиннер на весь экран.
- **Empty:** иконка (lucide, приглушённого цвета `text-muted-foreground`) + текст из `TZ.md` (раздел 15) + при необходимости кнопка действия ("Перейти до календаря").
- **Error:** иконка предупреждения + текст + кнопка "Спробувати ще раз" (`variant="outline"`, не основной красный — ошибка не должна визуально соревноваться с primary-действиями).
- **Success:** toast (shadcn `Toast`/`Sonner`) на 3 секунды, не модальное окно — не блокировать дальнейшее взаимодействие.

---

## 9. Как это связано с Cursor

1. Блок из раздела 1 — буквально скопировать в `app/globals.css` один раз, в начале Phase 1 (см. `Development_Roadmap.md`, задача 6). После этого все shadcn-компоненты автоматически используют эти цвета — отдельно прописывать цвета в каждом компоненте не нужно и не следует.
2. Короткая версия этих токенов уже включена в `.cursor/rules/design-system.mdc` — она грузится в каждый запрос к агенту автоматически, не нужно каждый раз напоминать про палитру.
3. Для конкретных сложных компонентов (например, точная раскладка карточки пилота) можно прикрепить скриншот-референс прямо в чат Cursor — текстовые токены задают систему, картинки уточняют конкретную компоновку.

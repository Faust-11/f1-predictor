# 1. Общая информация

## Цель проекта

Разработать современный веб-сервис для прогнозирования результатов этапов чемпионата Formula 1.

Пользователь должен иметь возможность:

- просматривать календарь сезона;
- просматривать результаты уже завершённых этапов;
- делать прогнозы на квалификацию;
- делать прогнозы на гонку;
- просматривать и редактировать свои прогнозы (до дедлайна);
- видеть рейтинг пользователей.

Интерфейс полностью на украинском языке.

Сайт должен корректно работать на ПК, планшетах и мобильных устройствах (от 320px).

> Это фан-проект, не имеющий официальной связи с Formula 1. Логотип, название и визуальный стиль должны быть **оригинальными**, вдохновлёнными стилистикой F1 (цвета, динамика), но не копировать официальный логотип/бренд-ассеты буквально.

# 3. Идентификация пользователя

В проекте **нет регистрации, email и паролей** — это осознанное решение для MVP, чтобы не утяжелять проект полноценной авторизацией.

**Механизм:**

1. При первом визите middleware генерирует анонимный `user_id` (UUID v4) и кладёт его в httpOnly cookie на 1 год.
2. Все прогнозы, история и место в рейтинге привязаны к этому `user_id`, а не к имени.
3. Имя ("Ваше ім'я") — это просто отображаемое поле, привязанное к `user_id`, его можно менять в профиле.
4. Если пользователь чистит cookies или меняет устройство — это новый анонимный профиль. Это ожидаемое и принятое ограничение MVP, не баг.

Полноценная авторизация (email/OAuth) — кандидат на пост-MVP, если понадобится синхронизация между устройствами.

# 5. Структура главной страницы

**Верхняя панель:** логотип "F1 Predictor", меню (Главная / Календарь / Рейтинг / Мої прогнози), переключатель темы.

**Баннер:** заголовок сезона.

**Ближайший этап:** название трассы, страна, дата квалификации, дата гонки, кнопка "Зробити прогноз" — кнопка ведёт на квалификацию, если прогноз на неё ещё не закрыт, иначе сразу на гонку.

**Далее:** последние результаты → календарь сезона (все этапы).

---

# 6. Календарь сезона

Получать автоматически через API. Для каждого этапа: название, страна, дата, статус (`upcoming` / `live` / `completed`).

После завершения этапа автоматически появляются результаты (см. раздел 13 про обновление данных).

**Error state:** если API недоступен и в базе нет свежих данных — показывать `Дані тимчасово недоступні. Спробуйте оновити сторінку пізніше.` с кнопкой "Оновити", а не пустую страницу.

# 7. Страница квалификации

## UX-сценарий

1. Пользователь выбирает режим: **Подиум** (P1-P3) или **TOP-10** (P1-P10).
2. Каждая позиция — пустой слот с подписью ("P1", "P2"...).
3. Клик по слоту → открывается модальное окно выбора пилота (раздел 9).
4. После выбора карточка пилота плавно "перелетает" в слот (Framer Motion).
5. Уже выбранный пилот помечается как недоступный в модальном окне выбора для других слотов — один пилот не может занимать две позиции одновременно.
6. Слот можно очистить и выбрать другого пилота повторно, до момента сохранения и до дедлайна.

## Бизнес-правило: дедлайн

За **5 минут** до начала квалификации кнопка "Зробити прогноз" исчезает и заменяется на "Переглянути результати" (если результаты уже есть) либо на "Прогнозування закрито" (если результатов пока нет).

Это правило проверяется **и на клиенте (UX), и на сервере** (см. `Architecture.md`, раздел "Безопасность") — скрытие кнопки само по себе не защита.

---

# 8. Страница гонки

Аналогична квалификации (Подиум / TOP-10, те же правила дедлайна — 5 минут до старта гонки).

Дополнительно: раздел **"Яка команда не фінішує?"** — карточки всех команд сезона, выбирается ровно одна, либо вариант **"Усі боліди фінішують"**. Выбор взаимоисключающий: выбор команды снимает выбор "усі фінішують" и наоборот.

---

# 9. Выбор пилота / команды

Модальное окно со списком пилотов: фото, имя, фамилия, трёхбуквенный код, логотип команды. Есть строка поиска по имени/коду.

После выбора — карточка занимает позицию, окно закрывается. Повторный выбор уже занятого пилота недоступен (визуально затемнён + disabled).

---

# 10. Сохранение и редактирование прогноза

После заполнения всех обязательных слотов — поле "Ваше ім'я" (если профиль анонимный без имени) → кнопка "Зберегти прогноз" → данные сохраняются, показывается подтверждение (toast/модалка "Прогноз збережено!").

**Редактирование:** до дедлайна пользователь может зайти на ту же страницу прогноза и изменить любую позицию — повторное сохранение **обновляет**, а не дублирует запись (один прогноз = одна строка на пользователя/этап/тип, см. ограничение `UNIQUE` в Architecture.md).

После дедлайна форма становится недоступной для записи (read-only просмотр того, что было сохранено).

---

# 11. Результаты этапа

После окончания этапа (статус становится `completed` по данным синхронизации) автоматически отображаются: результаты квалификации, результаты гонки, очки пользователя за этот этап, изменение его места в рейтинге.

**Empty state:** если этап ещё не завершён — `Результати з'являться після завершення етапу.`

---

# 12. Рейтинг пользователей

Таблица: имя, очки, количество этапов, количество точных попаданий, средняя точность.

**Тай-брейк** (при равенстве очков): сравнивать по количеству точных подиумов → затем по дате регистрации (раньше = выше).

**Empty state:** до первого завершённого этапа сезона — `Рейтинг з'явиться після першого етапу.`

---

# 13. История пользователя

Все прогнозы пользователя, полученные очки, результаты соответствующих этапов.

**Empty state:** `Ви ще не зробили жодного прогнозу. Перейдіть до найближчого етапу.`

# 14. Система начисления очков

> Это бизнес-логика, которая раньше допускала два толкования. Ниже — зафиксированный вариант. Если не согласны с конкретными числами или механикой — поправьте здесь, а не оставляйте Cursor решать самому.

## За каждую позицию (одинаково для квалификации и гонки)

| Позиция | Условие | Очки |
|---|---|---|
| P1 | точное совпадение | 10 |
| P2 | точное совпадение | 8 |
| P3 | точное совпадение | 6 |
| P4–P10 | точное совпадение позиции | 3 за каждую |

Это **точное совпадение слота**, а не просто попадание пилота в топ-10 без учёта позиции.

## Бонусы

- Весь подиум (P1+P2+P3 верны) — **+10** сверху
- Вся десятка (P1–P10 все верны) — **+30** сверху

## Сход с трассы (только гонка)

- Выбрана команда → **+10**, если хотя бы один болид этой команды не финишировал
- Выбрано "Усі боліди фінішують" → **+10**, если ни один болид не сошёл

Очки начисляются автоматически после появления официальных результатов (статус этапа → `completed`).

# 15. Обработка ошибок

| Ситуация | Что показывать |
|---|---|
| API недоступен, нет кэша | "Дані тимчасово недоступні. Спробуйте оновити сторінку пізніше." + кнопка обновления |
| Ошибка сохранения прогноза | "Не вдалося зберегти прогноз. Спробуйте ще раз." + повтор без потери введённых данных |
| Попытка сохранить после дедлайна | "Час для прогнозу на цей етап минув." |
| Сетевая ошибка при загрузке списка пилотов | Скелетон → при таймауте — сообщение об ошибке с retry |

# 16. Админ-панель

Доступ защищён паролем через переменную окружения (без полноценной системы пользователей — админка предполагается для одного человека, разработчика проекта).

Возможности:

- Просмотр журнала синхронизации с API (успех/ошибка, время) и кнопка "Синхронізувати зараз"
- Ручное изменение статуса этапа (upcoming / live / completed)
- Ручной ввод и корректировка результатов квалификации/гонки — на случай сбоя API или новых команд, по которым API ещё не отдаёт данные
- Ручной запуск пересчёта очков для конкретного этапа
- Просмотр списка пользователей и их прогнозов (read-only)

# 17. API и обновление данных
